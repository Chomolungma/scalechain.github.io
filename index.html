Problem

The number of Bitcoin transactions are growing over time, but Bitcoin full nodes require to keep all transactions from genesis block. The size of blockchain is also growing over time. Increasing block size of Bitcoin is not an ultimate solution to this problem, as Bitcoin core still needs to keep all transactions data in a node.

Possible Solutions

There are three ways to solve this issue.

  1. Reduce database size for a node by data compression techniques.
  2. Reduce the transactions on blocks using Sidechain, and off-chain transactions.
  3. Distribute data across nodes. ex> Tree-chain by Peter Todd.

In case you have an idea, feel free to visit the Bitcoin Scalability forum.
